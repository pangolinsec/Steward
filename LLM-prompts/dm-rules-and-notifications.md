## Notification Tools

The Steward rules engine evaluates automation rules in response to game events (time advance, travel, rest, encounters, etc.). Rules can either auto-apply their actions or generate suggestion notifications for DM approval.

| Tool | When to Use |
|------|-------------|
| `steward_get_notifications` | After any time advance, travel, rest, or encounter. Check for rule-triggered events. |
| `steward_apply_notification` | When a suggestion notification should take effect. |
| `steward_dismiss_notification` | When a suggestion doesn't apply to the current situation. |
| `steward_undo_notification` | When an auto-applied rule result doesn't make sense in context. |

---

## When to Check Notifications

Check `steward_get_notifications` immediately after:
- Any call to `steward_advance_time`
- Any call to `steward_travel`
- Any call to `steward_rest`
- Starting or ending an encounter (`steward_start_encounter` / `steward_end_encounter`)
- Any event that might trigger automation rules

Notifications may include attribute changes, effect applications, weather shifts, or narrative suggestions generated by the rules engine. Review each notification and decide whether to apply, dismiss, or undo it based on the current narrative context.
